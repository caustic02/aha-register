# Decision: Upgrade to Expo SDK 55

> Date: 2026-03-17
> Status: LOCKED
> Author: Michael Tauschinger-Dempsey + Claude Chat
> Scope: aha! Register mobile app

## Decision

Upgrade aha! Register from Expo SDK 52 (React Native 0.77, November 2024) to Expo SDK 55 (React Native 0.83, February 2026) before beginning the April Berlin demo build.

## Context

Register is a greenfield project with zero production users and zero legacy code. SDK 52 is three major versions behind the current stable release. The upgrade path from 52 to 55 crosses three breaking-change boundaries (53, 54, 55), including the mandatory New Architecture migration and the expo-file-system API replacement.

The alternative was to stay on SDK 52 for the demo and upgrade after Berlin. This was rejected because:

1. **No migration burden.** Register has no users, no production builds, no legacy native modules. The arguments for conservative upgrades apply to established apps, not greenfield.
2. **SDK 55 is the foundation.** Every library we'll adopt (Reanimated v4, FlashList v4, the new expo-file-system class API) requires or is optimized for SDK 55. Building on SDK 52 means building on deprecated APIs.
3. **New Architecture is mandatory going forward.** SDK 54 was the last version that allowed disabling it. Building new code on the Legacy Architecture and then migrating later is wasted work.
4. **SDK 56 arrives in May/June 2026.** If we stayed on SDK 52, we'd be four versions behind by demo time. Starting on the current stable means we're at most one version behind when SDK 56 ships, and that upgrade will be incremental.
5. **Expo Router v7** in SDK 55 introduces the new `/src/app` folder structure and Native Tabs API. Starting with this structure avoids a folder reorganization later.

## What Changes

| Component | SDK 52 (current) | SDK 55 (target) |
|---|---|---|
| React Native | 0.77 | 0.83 |
| React | 18.x | 19.2 |
| Architecture | Legacy (optional New) | New Architecture only |
| expo-file-system | Legacy class API | New class API (legacy removed) |
| Expo Router | v4 | v7 |
| Reanimated | v3 | v4 (New Arch only) |
| Hermes | v0.x | v1 (optional, build from source) |
| Minimum Node.js | 18.x | 20.19.x |
| Minimum Xcode | 16.0 | 16.1 |
| Android compileSdk | 35 | 36 |

## Migration Strategy

**Do not upgrade incrementally (52 > 53 > 54 > 55).** Because Register is greenfield with minimal native dependencies, the fastest path is:

1. Create a fresh SDK 55 project via `npx create-expo-app@latest --template default@sdk-55`
2. Copy application code (app/, src/, db/, services/, utils/, i18n/, contexts/) into the new project
3. Fix import paths (expo-file-system > new API, Expo Router v7 conventions)
4. Run `npx expo-doctor` to validate all dependencies
5. Build and test on Android emulator
6. Verify: camera capture, SQLite operations, EXIF extraction, SHA-256 hashing, i18n

This is a rebuild-in-place, not a migration. Estimated effort: one CC Opus session.

## Risks

- **Third-party library compatibility.** Some libraries may not support New Architecture. Mitigation: Register uses almost exclusively expo-* packages, which all support New Architecture as of SDK 53. Run `npx expo-doctor` to flag any issues.
- **React 19.2 breaking changes.** Concurrent features, useTransition behavior changes. Mitigation: Register's codebase is small and doesn't use advanced React patterns yet.
- **Hermes v1 is optional.** Don't enable it for the demo. It requires building React Native from source, which adds significant build time. Evaluate after Berlin.

## Consequences

- All future CC and Cursor prompts must target SDK 55 APIs
- expo-file-system imports use the new class API (not expo-file-system/legacy)
- newArchEnabled is removed from app.json (it's always on in SDK 55)
- Expo Go is limited in SDK 55; use development builds for testing
- The Android emulator configuration may need updating (API 36 target)
