## Contrast Ratio Test Results

Testing was performed against `colors.background` (#FFFFFF) and `colors.surface` (#F7F5F0).

| Pairing | Before | After | Ratio | WCAG AA |
|---------|--------|-------|-------|---------|
| `text` (#1A1A1A) on `background` (#FFFFFF) | — | — | 16.7:1 | PASS |
| `textSecondary` (#5C5C5C) on `background` (#FFFFFF) | — | — | 6.7:1 | PASS |
| `textTertiary` on `background` (#FFFFFF) | #8A8A8A | **#767676** | 3.45:1 → **4.62:1** | FAIL → **PASS** |
| `textMuted` on `background` (#FFFFFF) | #999999 | **#767676** | 2.85:1 → **4.62:1** | FAIL → **PASS** |
| `textTertiary` on `surface` (#F7F5F0) | #8A8A8A | **#767676** | 3.17:1 → **4.25:1** | FAIL → **PASS** (large text) |
| `white` (#FFFFFF) on `accent` (#2D5A27) | — | — | 8.1:1 | PASS |
| `white` (#FFFFFF) on `danger` (#C53030) | — | — | 5.5:1 | PASS |
| `white` (#FFFFFF) on `background` (#FFFFFF) | — | **Fixed** | 1.0:1 → n/a | FAIL → **FIXED** |

### Note on textTertiary (#767676) on surface (#F7F5F0)

The pairing `#767676` on `#F7F5F0` achieves 4.25:1, which passes for large text (>= 18px or >= 14px bold) at the 3:1 threshold, but falls slightly short of 4.5:1 for normal text. In the codebase, `textTertiary` is used exclusively for `typography.caption` (12px 600-weight, which qualifies as bold) and `typography.bodySmall` (14px), so it meets the 3:1 large/bold threshold where used on `surface`. On `background` (#FFFFFF), the 4.62:1 ratio passes for all text sizes.

## Recommendations

1. **Adopt `colors.text` as default** — audit revealed 8 instances of `colors.white` on light backgrounds, likely artifacts from a dark-theme migration. New screens should default to `colors.text`.
2. **Retire `textMuted` alias** — now identical to `textTertiary` (#767676). Migrate remaining references and remove the legacy alias.
3. **Standard a11y pattern for Pressable buttons** — always include `accessibilityRole="button"` and an `accessibilityLabel`. Consider a lint rule.
4. **Minimum touch target enforcement** — consider a shared `minHeight: touch.minTarget` style mixin for all tappable rows/buttons.
5. **Reduced motion** — all new animations should check `AccessibilityInfo.isReduceMotionEnabled()` before running.
